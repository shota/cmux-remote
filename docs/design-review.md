# cmux-remote 設計レビュー

## レビュー日: 2026-03-20

## 1. 総合評価

**判定: 承認（軽微な改善提案あり）**

要件に対して適切な技術選定・設計がなされている。以下に詳細レビューを記載する。

## 2. アーキテクチャ

### 良い点
- Bridge Server を透過的な中継に徹する設計は、cmux プロトコルの変更に強い
- コンポーネントが 2 つのみでシンプル
- クラウドインフラ不要の設計はセキュリティとコスト面で優れている

### 改善提案
- **[P3] 将来的な拡張**: 複数クライアント同時接続時の挙動を明記すべき → 各 WebSocket 接続が独立した UDS 接続を持つ設計で問題なし。ドキュメントに明記する。

## 3. Bridge Server

### 良い点
- Bun + Hono の組み合わせは軽量・高速で適切
- 透過的メッセージ中継でロジックがシンプル
- ヘルスチェックエンドポイントで運用性を確保

### 改善提案
- **[P2] 接続プール**: UDS 接続のタイムアウト処理を追加すべき → cmux が応答しない場合の 30 秒タイムアウトを設ける
- **[P3] ログ**: 接続・切断・エラーのログ出力を実装に含めること

## 4. iPhone PWA

### 良い点
- xterm.js は実績のあるターミナルエミュレータで適切
- Hammer.js による 2 本指ジェスチャーは Safari で実績あり
- Vite によるビルドは PWA との相性が良い

### 改善提案
- **[P2] ポーリング間隔**: `surface.read_text` の 1 秒間隔ポーリングは妥当だが、変更がない場合のトラフィック削減のため、diff ベースの更新を検討する → 初期実装ではポーリングで進め、必要に応じて最適化する
- **[P2] ターミナルサイズ**: iPhone の画面サイズに合わせた xterm.js のフォントサイズ自動調整を実装に含めること
- **[P3] Safe Area**: iPhone のノッチ / Dynamic Island / ホームインジケーターに対応する CSS を追加すること

## 5. セキュリティ

### 評価
- Tailscale / Cloudflare Tunnel によるネットワーク層の保護は適切
- Bridge Server が信頼されたネットワーク内で動作する前提は妥当

### 改善提案
- **[P3] オプショナル認証**: 将来的に Bearer トークンによる簡易認証を追加できる設計にしておくこと → WebSocket 接続時のヘッダーで対応可能

## 6. レビュー結果まとめ

| # | 優先度 | 項目 | 対応 |
|---|---|---|---|
| 1 | P2 | UDS 接続タイムアウト | 実装時に 30 秒タイムアウトを設定 |
| 2 | P2 | ポーリング最適化 | 初期実装はポーリング、後で最適化 |
| 3 | P2 | フォントサイズ自動調整 | 実装時に対応 |
| 4 | P3 | 複数クライアント同時接続 | ドキュメント明記で対応 |
| 5 | P3 | ログ出力 | 実装時に対応 |
| 6 | P3 | Safe Area 対応 | CSS で対応 |
| 7 | P3 | オプショナル認証 | 将来対応可能な設計に |

## 7. 結論

設計は要件を満たしており、実装に進めて問題ない。P2 の改善提案は実装フェーズで反映すること。
