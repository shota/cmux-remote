## 2. サーバー実装レビュー

### 2.1 cmux-client.ts
- **良い点**: newline 区切り JSON の正しいバッファリング処理。30 秒タイムアウト実装済み（設計レビューの P2 指摘に対応）。
- **良い点**: 接続ライフサイクル（connect/disconnect/checkConnection）が明確に分離されている。
- **確認済み**: ソケットパスのデフォルト値が `~/Library/Application Support/cmux/cmux.sock` で要件通り。

### 2.2 ws.ts
- **良い点**: WebSocket 接続ごとに独立した CmuxClient を生成する設計（設計書 2.3.1 に準拠）。
- **良い点**: 切断時のクリーンアップが実装されている。
- **良い点**: cmux 接続失敗時のエラー通知と WebSocket クローズが実装されている。

### 2.3 index.ts
- **良い点**: WebSocket upgrade と Hono ルーティングの分離が明確。
- **良い点**: 環境変数 PORT によるポート設定が可能。
- **確認済み**: 静的ファイル配信パスが `../client/dist` で正しい。

### 2.4 health.ts
- **良い点**: cmux ソケットの接続テスト結果を返す実装。uptime 情報も含まれる。

## 3. クライアント実装レビュー

### 3.1 lib/cmux-rpc.ts
- **良い点**: RPC のリクエスト生成と型定義が分離されている。
- **良い点**: インクリメンタル ID 生成で重複なし。

### 3.2 hooks/useWebSocket.ts
- **良い点**: 指数バックオフ再接続（1s, 2s, 4s, ... max 30s）が設計通り実装されている。
- **良い点**: `onMessageRef` パターンでコールバックの stale closure 問題を回避。

### 3.3 hooks/useCmux.ts
- **良い点**: RPC のリクエスト/レスポンス対応管理（pending Map + タイムアウト）が実装されている。
- **良い点**: `navigateWorkspace` / `navigatePane` で循環ナビゲーションを実装。
- **良い点**: 10 秒 RPC タイムアウトが設計通り。

### 3.4 hooks/useGesture.ts
- **良い点**: Hammer.js の 2 本指パン検出で、方向判定ロジックが正しい。
- **良い点**: 触覚フィードバック対応。

### 3.5 components/Terminal.tsx

### 3.6 components/Drawer.tsx

### 3.7 App.tsx
